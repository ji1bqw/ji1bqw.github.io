ascpserialdモニター

--- Internet Linked Radio Amateur ---

当局ではDV Dongleを24時間運転のLinuxサーバーに接続し，複数のWindows PCからリモートアクセスして使っていることは先日書いた通りです。

Linuxサーバー側ではascpseriald.lnxというプロセスをバックグラウンドで走らせているのですが，このプロセスはネットワークが切れたりすると終了してしまいます。時々リモートからDV Dongleをアクセスしようとして失敗するときには，このプロセスが動いていないことが主な原因です。

そこで，このプロセスを監視し，もし動作していないようなら再起動するようなシェルスクリプトを，cronから5分おきに動かすようにしました。

モニタースクリプトの基本は以下のように極単純なものです。

#!/bin/sh
#
# ascpseriald.mon
#
# Check to see if ascpseriald.lnx is running, and if not launch it.
# Run from crontab.
# JI1BQW

DVTool=/root/DVTool

if ! pidof ascpseriald.lnx ; then
   $DVTool/ascpseriald.sh > /dev/null 2>&1 &

exit 0

実際にはこれにログを書いたり，再起動を確認するようなコードを追加してますが，基本的には上のコードだけで大丈夫です。

そして，crontab (root) のエントリーは以下のようになります。

*/5 * * * * /root/DVTool/ascpseriald.mon > /dev/null 2>&1

これでさらに安定してDV Dongleを利用することができるようになりました。
