モニターモード

--- Internet Linked Radio Amateur ---

今日は小さな、しかし結構便利な機能がDave C. によって追加されました。

それは monitor only mode と呼ばれています。その名のとおりモニターのみ、すなわち、こちら側ではネットワークの向こう側にいる接続相手局あるいはリフレクターからの音声を送信するだけで、こちら側のローカル局が送信してもデータ送出されず相手にはこちらの音声は届きません。

この機能は imike を別のモード（-h フラグをつけて実行し、PCのスペースバーを押すことで送信するモード）で実行することで、今までも実現できていました。しかし、一度接続した後、imikeプロセスをKillして、コマンドラインから -h フラグと相手のIPアドレスを指定してimikeを再起動する必要がありました。

これを今回、$LOCAL/monitor_onlyというファイルが存在する時には、IRLPのプログラムが自動的に imike に　-h フラグを付けて接続してくれるように変更が加えられました。

よって、monitor only mode にするには、
    touch $LOCAL/monitor_only
を実行してから接続に行けばOKです。

その接続が切断されると、自動的に$LOCAL/monitor_onlyファイルは削除されますので、次の接続は monitor only mode にはなりません。

###それでは、ずっと monitor only mode にしておきたいときは、どうしますか。IRLPの機能を使う、Linuxの機能を使うなどいくつかやり方はありそうですね。

ローカルの音声が相手に届かないと言っても、imikeの機能でそうしているだけですので、DTMFは引き続き機能します。したがって、DTMFから monitor only mode に切り替えることも可能です。

IRLP自動アップデートがかかっていれば、すでに更新されているはずです。どのファイルが、どのように変更されているかを見るのも楽しいものです。
