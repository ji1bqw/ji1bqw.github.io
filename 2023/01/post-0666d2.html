Some notes for installing VarAC on Raspberry Pi

--- Internet Linked Radio Amateur ---

Unlike VARA HF Modem, installing and running VarAC on Raspberry Pi is tricky.  Here are some notes I took when I successfully had it run.  From my 10+ different attempts, this is the most reliable way to install VarAC and get it running.

That being said, it should be taken as nothing more than an experiment.  I must admit I use Linux Mint for daily operations.

Prerequisites

  • RPi 4B with Raspberry Pi OS 32bit.  KM4ACK's "build-a-pi" is a very good option to consider for a Ham purpose.

Raspberry Pi OS 64bit may work, but I have yet to confirm it.

Run WheezyE's install script with vara_only option

https://github.com/WheezyE/Winelink

This will install box86, wine, winetricks, VARA HF and FM Modems and winklink-related components.   If you only need VARA but no other winlink-related components, specify the vara_only option when you run the install script.

$ curl -O https://raw.githubusercontent.com/WheezyE/Winelink/main/install_winelink.sh && bash install_winelink.sh vara_only

Replace PDH (Performance Data Helper) in the wine prefix

$ cd $HOME/Downloads

$ wget http://files.k6eta.com/pdh.dll.zip

$ unzip pdh.dll.zip

$ rm pdh.dll.zip

$ mv pdh.dll $HOME/.wine/drive_c/windows/system32

By now, you should be able to run VARA Modem standalone.

Install wine-mono

$ wget -q -P ~/.cache/wine https://dl.winehq.org/wine/wine-mono/7.2.0/wine-mono-7.2.0-x86.msi

$ wine msiexec /i ~/.cache/wine/wine-mono-7.2.0-x86.msi

Now winecfg shows something like below.

Install VarAC

Unzip VarAC.zip under "~/.wine/drive_c/"

It is highly recommended that VarAC.ini be copied from another working environment or carefully created by hand. This could reduce the likelihood of failures at startup.

Don't forget to set "LinuxCompatibleMode=ON" in VarAC.ini.

Run VarAC.exe

$ wineserver -k

$ wine VarAC.exe

You will see tons of errors and warning messages, but let it go.

After VarAC and VARA Modem are launched, in many cases TCP LED of VARA Modem stays red. Repeat "DISCONNECT MODEM"/"CONNECT MODEM" on VarAC UI until the LED turns green.

Just in case, let's run "wineserver -k" to kill all wine related processes before launching VarAC.

Turn off Narrator on VarAC Settings
"Setting" -> "Appearance and Sounds" -> Uncheck all options at "Narrator (Text-to-Speech)"
This is important.  If you don't disable this, VarAC is most likely to freeze or crash when it narrates (e.g. when received a CQ call).
